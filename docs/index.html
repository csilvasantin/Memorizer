Guía de instalación

Puesta en marcha

Memorizer puede ejecutarse con Docker (recomendado) o directamente en tu entorno Python.

1

Clona el repositorio

bash
$ git clone https://github.com/csilvasantin/Memorizer.git
$ cd Memorizer
2

Crea tu archivo de entorno

Copia el archivo de ejemplo y rellena tus credenciales.

bash
$ cp .env.example .env
$ nano .env   # o tu editor preferido
.env
# Telegram
TELEGRAM_BOT_TOKEN=123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11
TELEGRAM_GROUP_ID=-1001234567890

# Anthropic
ANTHROPIC_API_KEY=sk-ant-api03-...

# Opcional
CLAUDE_MODEL=claude-sonnet-4-20250514
DATABASE_PATH=data/memorizer.db
3

Lanza con Docker Compose

bash
$ docker compose up -d

# Ver logs en vivo
$ docker compose logs -f

# Detener
$ docker compose down
4

Añade el bot a tu grupo

Busca @Memorizer2Bot en Telegram, añádelo al grupo como administrador para que pueda leer mensajes y reaccionar a ellos.

ℹ️
¿Qué ID tiene mi grupo? Reenvía cualquier mensaje del grupo a @userinfobot en Telegram para obtener el ID del chat. Los grupos suelen tener ID negativo (–100…).
1

Requisitos previos

Necesitas Python 3.12 o superior instalado en tu sistema.

bash
$ python --version   # Python 3.12+
2

Clona e instala dependencias

bash
$ git clone https://github.com/csilvasantin/Memorizer.git
$ cd Memorizer
$ python -m venv .venv
$ source .venv/bin/activate   # Windows: .venv\Scripts\activate
$ pip install -r requirements.txt
3

Configura el entorno

bash
$ cp .env.example .env
$ nano .env   # Añade tus tokens
4

Ejecuta el bot

bash
$ python -m src.bot
⚠️
Mantener activo: Para producción usa Docker o un servicio del sistema (systemd/supervisor) para que el bot no se detenga al cerrar la terminal.

→

→

→

→

Referencia de comandos

Comandos del bot

Todos los comandos se envían directamente en el chat del grupo o en conversación privada con el bot.

/buscar
Busca en tus memorias guardadas.
Combina búsqueda de texto completo (FTS5) con un LLM para responder de forma contextual. Si no hay resultados exactos, usa las memorias más recientes como contexto de fallback.
/buscar proyecto Alpha
/resumen
Genera un resumen de los últimos N días.
Por defecto muestra los últimos 7 días. Pasa un número para cambiar el período. El resumen está organizado por categorías y es generado por Claude.
/resumen 30
/stats
Muestra estadísticas de la base de datos.
Total de memorias guardadas, desglose por categoría y por fuente detectada (LinkedIn, WhatsApp, web, etc.).
/stats
/help
Muestra la ayuda del bot.
Lista todos los comandos disponibles con una breve descripción. También disponible con /start al iniciar conversación con el bot.
/help
💡
Captura automática: No necesitas usar ningún comando para guardar información. Cada mensaje que envíes al grupo se captura, clasifica y almacena automáticamente. El bot reacciona con el emoji de la categoría detectada para confirmar que procesó el mensaje.

Arquitectura

Módulos del sistema

Memorizer está organizado en módulos desacoplados. Cada fichero tiene una responsabilidad clara.

🤖
src/bot.py
Punto de entrada · Handlers de Telegram
CORE

Orquesta toda la aplicación. Registra los handlers de comandos y mensajes, inicializa la base de datos al arrancar y coordina el flujo: recepción → clasificación → almacenamiento → reacción.

  • handle_message() Captura todos los mensajes del grupo, delega en classifier y storage
  • cmd_buscar() Handler de /buscar — llama a answer_query()
  • cmd_resumen() Handler de /resumen [dias] — llama a generate_summary()
  • cmd_stats() Handler de /stats — muestra conteos por categoría y fuente
  • _get_content_type() Normaliza el tipo de contenido del mensaje (text, photo, video…)
✨
src/classifier.py
Clasificación IA · Extracción de entidades
AI

Envía cada mensaje a Claude con un prompt estructurado y parsea la respuesta JSON. Incluye detección local de fuente como fallback y extracción de URLs con regex. Si la API falla, devuelve una clasificación básica sin romper el flujo.

  • classify_message() Función principal — llama a Claude, parsea JSON, maneja errores
  • detect_source() Detecta la plataforma de origen por keywords (linkedin, whatsapp, twitter…)
  • extract_urls() Extrae URLs del texto con regex
🔧
El prompt le pide a Claude que devuelva: category, summary, entities, urls y source_hint en un único bloque JSON.
🗄️
src/storage.py
Persistencia · SQLite con FTS5
DB

Gestiona la base de datos SQLite con una tabla principal memories y una tabla virtual FTS5 memories_fts sincronizada mediante triggers. Todas las operaciones son asíncronas gracias a aiosqlite.

  • init_db() Crea las tablas y la tabla virtual FTS5 si no existen
  • save_memory() Inserta una memoria; los triggers actualizan el índice FTS automáticamente
  • search() Busca con MATCH en FTS5 ordenando por relevancia (rank)
  • get_recent() Últimas N memorias ordenadas por fecha
  • get_by_category() Filtra memorias por categoría
  • get_by_date_range() Memorias entre dos fechas (usado por /resumen)
  • get_stats() Totales y agrupaciones por categoría y fuente
🔍
src/query.py
Motor RAG · Respuestas con contexto
RAG

Implementa el patrón RAG (Retrieval-Augmented Generation): busca las memorias más relevantes en SQLite, las formatea como contexto y las pasa a Claude para que genere una respuesta natural. Si la búsqueda exacta no produce resultados, usa las memorias más recientes como fallback.

  • answer_query() Busca en FTS5, construye contexto y genera respuesta con Claude
  • generate_summary() Recupera memorias del período y genera resumen organizado por categorías
  • get_stats_text() Formatea las estadísticas de storage como texto legible
  • _format_results() Serializa resultados SQL en texto para el contexto del prompt
⚙️
src/config.py
Configuración central · Variables de entorno
CONFIG

Carga todas las variables de entorno desde .env usando python-dotenv y las expone como constantes tipadas. También define las listas de categorías y las palabras clave para detección de fuentes.

  • CATEGORIES Lista de categorías válidas para clasificación
  • SOURCE_KEYWORDS Dict con palabras clave por plataforma para detección local

Esquema de base de datos

SQLite — tabla memories
CREATE TABLE memories (
  id                  INTEGER  PRIMARY KEY AUTOINCREMENT,
  telegram_message_id INTEGER  UNIQUE,
  chat_id             INTEGER,
  author              TEXT,
  content             TEXT     NOT NULL,
  content_type        TEXT     DEFAULT 'text',
  source              TEXT     DEFAULT 'unknown',
  category            TEXT     DEFAULT 'otro',
  summary             TEXT,
  entities            TEXT,    -- JSON array
  urls                TEXT,    -- JSON array
  created_at          TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  processed_at        TIMESTAMP
);

-- Tabla virtual FTS5 sincronizada por triggers
CREATE VIRTUAL TABLE memories_fts USING fts5(
  content, summary, entities, author, source, category,
  content='memories', content_rowid='id'
);
Referencia de configuración

Variables de entorno

Toda la configuración se gestiona mediante el archivo .env en la raíz del proyecto.

Variable Descripción Por defecto Requerida
TELEGRAM_BOT_TOKEN Token del bot obtenido de @BotFather — Requerida
TELEGRAM_GROUP_ID ID del grupo de Telegram a monitorizar. Usa 0 para escuchar cualquier grupo. 0 Opcional
ANTHROPIC_API_KEY Clave API de Anthropic para acceder a Claude. Obtenida en console.anthropic.com — Requerida
CLAUDE_MODEL Identificador del modelo Claude a usar para clasificación y respuestas claude-sonnet-4-20250514 Opcional
DATABASE_PATH Ruta al archivo SQLite. Se crea automáticamente si no existe. data/memorizer.db Opcional
🔒
Seguridad: Nunca subas tu archivo .env al repositorio. El .gitignore ya lo excluye. Para producción usa secretos del sistema (variables de entorno del sistema operativo, Docker secrets, etc.).

Tecnologías

Stack tecnológico

Herramientas modernas seleccionadas por su fiabilidad, rendimiento y facilidad de despliegue.

🐍
Python 3.12
≥ 3.12
Lenguaje principal. Usa asyncio y sintaxis moderna de tipos (str | None, list[dict]).
✈️
python-telegram-bot
≥ 21.0
Framework asíncrono para la API de Telegram. Maneja polling, handlers y contexto de manera elegante.
🧠
Anthropic SDK
≥ 0.40.0
SDK oficial de Anthropic para acceder a Claude. Usa el modelo claude-sonnet-4 para clasificación y generación.
🗄️
SQLite + FTS5
Built-in
Base de datos local sin servidor con extensión FTS5 para búsqueda de texto completo con ranking por relevancia.
⚡
aiosqlite
≥ 0.20.0
Wrapper asíncrono de SQLite compatible con asyncio, evita bloquear el event loop durante operaciones I/O.
🐳
Docker
Compose v2
Contenedor basado en python:3.12-slim. El volumen ./data persiste la base de datos entre reinicios.
